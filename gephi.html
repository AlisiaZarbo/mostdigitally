The process

It is possible to consider this section as the last and conclusive work, made possible by the methods previously applied on this work. Thus, after have collected the different way with which every character is named inside the text (trough the Xpath queries), the occurrence of every character has been collected (trough the names collected). Pride and prejudice is a story builded on relations, of family, love and frienship type. Thus, it has been decided to provide to the reader/user an explicative graph where the most important relations are shown.

Data collection

The data collected have been taken by the results provided by the analysis done with Voyant. The datum about the occurrence of every character allow to establish a gerarchy of the characters; on their frequency, indeed, it depends the importance of the subject in story. Before to visualize the graph, it is necessary to fill a table (in this case it has been done in an external, csv, file) where every cell contains a node, to which an index is assigned. The other columns can be filled with all the values that could be necessary to characterize the data. Indeed, in this special case, a column has been dedicated to the frequency value of every index, represented by an integer. Another table, instead, has been dedicated to the edges, that in this case want to represent the relations between the characters of the history.

Layout choices

The column with the occurence of every character filling the cell node allowed to select in the "Appearence" section the size of the nodes depending on their integer value. So the nodes are now shown in their size depending on their importance inside the story. Every value is represented by a different color. The edges, instead, represent the relations between the characters; they where divided into three kinds: family, love and friendship; every kind is shown in a different color. Two different layouts have been provided, the first showing the labels (the names of the characters), the second without labels.
